# 提醒桥接收口优化记录

## 目标

在不改变现有提醒桥接能力的前提下，降低 `reminder_task_bridge` 的重复分支和重复日期计算，方便后续继续扩展自然语言提醒规则。

## 本轮优化方向

- 收口 `parse_due_at()` 的串行分支，改成统一的解析器链
- 抽出通用时间解析与标准化助手，避免多处重复校验 `hour/minute`
- 抽出月切换、指定日期构造、下一次月初/月末/季度末计算逻辑
- 收口循环任务描述和意图构造，减少重复 pattern search

## 预期收益

- 新增提醒规则时只需要补新的解析函数并挂到解析器链
- 日期类规则更容易复用，不再反复写“本月/下月/跨年”分支
- 单测和回归更稳定，后续更容易补边界场景
