# 群聊唤醒词误绕过秘书修复（2026-04-02）

## 背景

在群聊里，如果消息命中了 AstrBot 的 `wake_prefix`，`event.is_at_or_wake_command` 会被置为 `True`。  
旧逻辑下，AngelHeart 在群聊里会进一步要求：

- 必须是 `@自己`
- 或 `Reply自己`

否则就把这条消息视为“指令或 @他人”，直接忽略。

这会导致一个实际问题：

1. AstrBot 主链路已经被唤醒
2. AngelHeart 却没有缓存消息、没有秘书分析、没有注入当前轮上下文
3. 主模型只能依赖其他插件注入的记忆或人格信息回答
4. 于是容易把已经结束的旧话题重新翻出来，出现明显跑题

## 本次修复

群聊 `wake_prefix` 命中后，改成区分两类：

- 明确命令型唤醒
  - 当前事件已经命中了插件命令处理器 (`activated_handlers`)
  - 继续跳过 AngelHeart，避免和命令插件抢处理权
- 自然语言型唤醒
  - 没有命中命令处理器
  - 且消息仍有自然语言正文
  - 允许进入 AngelHeart 缓存和秘书链

## 预期效果

- `ai主机就是笑话` 这类自然语言唤醒，不会再绕过秘书
- 当前轮聊天记录会进入 AngelHeart 总账
- 秘书决策和上下文能正常注入主脑
- 减少“当前场景没上下文，主模型被旧记忆带偏”的突兀回复
