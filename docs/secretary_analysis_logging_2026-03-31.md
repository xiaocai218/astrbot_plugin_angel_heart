# 秘书分析结果日志调整记录（2026-03-31）

## 背景

当前秘书流程只在“开始调用 LLM 分析”时打印 `info`，但分析完成后缺少统一的结果日志。
这会导致排查时只能从前台执行链路间接判断秘书是否决定参与回复。

## 本次调整

1. 在 `Secretary.handle_message_by_state()` 中统一收口分析结果日志。
2. 每次秘书完成“是否参与回复”的判断后，都会打印一条 `info`。
3. 当秘书决定不参与时，日志会明确输出“不参与回复”以及原因。
4. 前台执行链路不再重复打印“决策为参与/不参与”，改为执行态日志，避免同一轮决策重复刷屏。

## 目标效果

- 便于直接从秘书日志观察决策结果。
- 减少排查时对前台执行日志的依赖。
- 保持各状态分支的日志格式一致。
